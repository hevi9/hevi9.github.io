Pelican usage notes

Install software

Install packages:

> pip3 install pelican markdown ghp-import watchdog

http://blog.getpelican.com/ pelican

https://pythonhosted.org/Markdown/ markdown

ghp-import is a python script that imports a given directory tree into local current git repository as gh-pages branch, which can later pushed into github.

watchdog

Setup blog directory

Make a directory or use existing one. ~/wrk/blog is used here and CWD there is assumed on later examples.

Initialize files:

> cd ~/wrk/blog
> mv .project .project-no
> pelican-quickstart
> mv .project-no .project

Move eclipse project file away (if any), pelican-quickstart confuses with it.

Run notes

First terminal:

> make serve

and open browser to http://localhost:8000

Second terminal:

> pelican -r

This run until there rst syntax error, then restart pelican -r again.

OR

Use file modify watch program to run pelican again, Makefile fragment:

rerun:
  watchmedo shell-command  -i "*output/*;*cache/*"  -W -R -c "make html" .

Beam up to github user pages

Makefile fragment (full https://github.com/hevi9/blog/blob/master/Makefile):

github: publish
  ghp-import -n $(OUTPUTDIR)
  git push -f git@github.com:$(GHP_USER)/$(GHP_USER).github.io.git gh-pages:master

Import output directory into local git repo as gh-pages branch and the push gh-pages into remote github user pages repo as master branch. Note: this uses -f force overwrite on push so existing remote files are destroyed which is ok on this case.

http://docs.getpelican.com/en/3.4.0/tips.html
